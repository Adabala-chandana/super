# Q02 — Annotations: null-field validator

Build a small validation utility driven by a custom annotation.

## Requirements

1. Define a custom annotation `@NotNull` that can be placed on fields (`@Target(ElementType.FIELD)`, runtime retention).
2. Implement `Validator.validate(Object obj)` that:
   - reflects over declared fields of `obj`'s class
   - for each field marked `@NotNull`, checks the value is non-null
   - collects failure messages like `"email must not be null"`
3. Demo on a POJO, e.g. `UserRegistration` with `name` and `email` both `@NotNull`.

## Constraints

- Must use your custom annotation (not Bean Validation libraries).
- Use reflection to read field values (set accessible if needed).
- Return a `List<String>` of errors (empty list means valid).

## Expected behavior

```text
UserRegistration bad = new UserRegistration(null, null);
List<String> errors = Validator.validate(bad);
// contains messages for name and email

UserRegistration ok = new UserRegistration("Ada", "ada@example.com");
Validator.validate(ok); // empty list
```
