# Q01 — Generics: type-safe repository

Implement a generic in-memory `Repository<T>` that stores entities by id without requiring casts at the call site.

## Requirements

1. `T` must expose an id via a method or interface (e.g. `Identifiable` with `Long getId()`).
2. Provide:
   - `void save(T entity)`
   - `Optional<T> findById(Long id)`
   - `List<T> findAll()`
3. Demonstrate usage with two different entity types (e.g. `User` and `Product`) in a small `main` or test — both repositories must be compile-time type-safe (no `ClassCastException`, no raw types).

## Constraints

- Use Java generics (Java 5+ style is fine).
- Do not use separate non-generic repositories for each entity.
- `findById` must return the concrete type for that repository (e.g. `Optional<User>`, not `Optional<Object>`).

## Expected behavior

```text
Repository<User> users = new Repository<>();
users.save(new User(1L, "Ada"));
User ada = users.findById(1L).orElseThrow();  // type is User, no cast

Repository<Product> products = new Repository<>();
products.save(new Product(10L, "Keyboard"));
Product p = products.findById(10L).orElseThrow();  // type is Product, no cast
```
